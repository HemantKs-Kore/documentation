Tokenizer
Tokenizer tokenizes the given user input into smaller tokens which are useful for parsing.
Stemmer
Stemmer will stem each eligible word removing the suffixes like ing, tion etc.
Case Sesitivity
Retain the case or convert to lowercase before querying
Entity Placeholder
Reduces the relvancy of entity values. Ex: "How do I transfer 100$ to John." Enabling this will remove relenacy of $100 and John.
Stopwords
Discard stop words present in user utterance for detecting intents.
Sentiment
Enable identification of Sentiment for the user input
Acronym Expanstion
Automatically expands local slangs like IMHO, DIY,TY etc into full words
Traits
Enable the identification of traits in user input
POS tagging
Use Parts of Speech information for Querying
Emoticon Expansion
Automatically expands emoticons. Can be useful to understand sentiment. In some case acknowledgements like thank you, "done" etc.
Negation Handling
Handles cases where user explicity mentions negation. Ex: I do not want to transfer funds, should not identify the intent to Transfer funds
